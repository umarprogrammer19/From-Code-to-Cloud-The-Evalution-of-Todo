# Data Model: User and Task Entities

This document defines the data models for the User and Task entities.

## User Entity

-   **Represents**: A user of the application.
-   **Managed by**: Better Auth
-   **Key Attributes**:
    -   `id`: string (Primary Key)
    -   `email`: string (unique)
    -   `name`: string
    -   `created_at`: datetime
-   **Validation Rules**:
    -   `email` must be a valid email format.
    -   `id` is generated by the authentication provider.

## Task Entity

-   **Represents**: A to-do item.
-   **Key Attributes**:
    -   `id`: integer (Primary Key)
    -   `title`: string
    -   `description`: string (optional)
    -   `completed`: boolean
    -   `user_id`: string (Foreign Key to User.id)
-   **Relationships**:
    -   Belongs to a User.
-   **State Transitions**:
    -   A task can be marked as `completed` or `uncompleted`.
